## Validation target

Workflows must:

1. check out the exact submitted head;
2. use read-only permissions and no persisted credentials;
3. install pinned dependencies;
4. require decision, inventory, currentness, roadmap, contributor, lifecycle, D4, D5, architecture, security, onboarding, and diagram surfaces;
5. run validators, hostile regressions, link checks, and `mkdocs build --strict`;
6. validate terminology, prohibited claims, authority separation, freeze and evidence boundaries, claim withdrawal, recovery, closure, and route consistency;
7. hash source and rendered evidence;
8. retain terminal-path evidence;
9. fail closed after evidence retention.

Passing validation proves construction only.

## Rollback criteria

Restore the last reviewed documentation state, preserve failed evidence, invalidate derived claims, and prohibit destructive history rewriting. Rollback must not resurrect revoked capabilities, withdrawn consent, stale credentials, superseded checkpoints, invalidated authority, or withdrawn public claims.
