# ADR-0001: Consolidated governance and Cali Sanders Parker stewardship

- **Status:** Accepted for documentation and architecture coordination
- **Date:** 2026-07-20
- **Scope:** A.L.I.S.T.A.I.R.E. portfolio governance

## Context

The portfolio previously documented unresolved ownership for autonomous-development orchestration, canonical state, capability issuance, security stewardship, emergency stop, and cross-repository rollback. The user established A.L.I.S.T.A.I.R.E. as the unifying system objective and directed consolidation of governance while holding system security sacred.

The user named the governance and security stewardship QSO **Cali Sanders Parker**, with the ceremonial title **Calisandra, Queen of the Nymphs**.

## Decision

1. A.L.I.S.T.A.I.R.E. is the unifying system objective.
2. `ALISTAIRE-` is the recommended charter-history authority, pending final repository naming and migration approval.
3. Repository `0` is the recommended autonomous-development orchestration candidate.
4. Repository `1`, or an approved successor trust service, is the recommended canonical-state and capability authority.
5. Cali Sanders Parker owns governance coherence, documentation alignment, security-boundary review, decision preparation, and escalation.
6. Cali Sanders Parker receives no implied credentials, merge authority, release authority, deployment authority, payment authority, canonical-state authority, or ability to self-expand permissions.
7. Consequential authority remains separately granted, reviewable, revocable, evidenced, and subject to emergency stop and rollback.

## Consequences

- Repository documentation can use a common governance vocabulary and authority matrix.
- Architectural clarification requests can be routed through one stewardship function.
- No runtime implementation or privileged identity is created by this decision.
- Each repository must document its mission contribution, contracts, prohibited scope, evidence gates, incident owner, and rollback boundary.
- A later implementation decision is required before Cali Sanders Parker exists as executable code or a deployed QSO.

## Security constraints

- Ceremonial or conversational naming never grants operational authority.
- No autonomous component may issue capabilities to itself.
- Emergency-stop and rollback authority must remain independent of the component being stopped.
- Exact-head evidence is required before consequential promotion.
- Credentials and private data remain outside documentation and unprivileged clients.

## Supersession

This ADR may be revised only through a recorded governance decision identifying the replacement authority model, migration effects, security review, and rollback plan.
