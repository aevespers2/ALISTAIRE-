A documentation-first coordination repository for an evidence-bounded, QSO-composed agent architecture.

## What is established

- Alistaire is intended to be described as a female-presenting agent system composed from interoperable Quantum State Objects.
- The first eligible candidate is documentation-only.
- Runtime implementation remains downstream of canonical-repository approval, an approved charter, and stable upstream QSO contracts.
- `ALISTAIRE-` and `Alistaire-agi` currently overlap in name and product identity; neither repository may treat that overlap as approval to supersede the other.

## What is not established

- AGI, consciousness, sentience, human equivalence, autonomous learning, or production readiness.
- A canonical repository, package name, release identity, migration, archive, or redirect decision.
- Approved subsystem ownership, contract versions, persistence policy, external-tool authority, credential use, network use, or deployment.
- Completed security, privacy, provenance, evaluation, rollback, or release evidence.

## Documentation-only architecture boundary

```text
portfolio/product directive
        ↓
repository consolidation inventory
        ↓
canonical-repository decision  ← explicit approval required
        ↓
product and architecture charter
        ↓
versioned QSO contracts and deterministic fixtures
        ↓
```

Everything below the canonical-repository decision remains blocked. Documentation may inventory, clarify, preserve provenance, expose contradictions, and prepare review material; it may not create implementation or operational authority.

## Contributing safely

While P0 is blocked, contributions are limited to documentation inventory, terminology clarification, provenance preservation, accessible diagrams, link validation, and evidence classification. Do not add runtime code, credentials, external data ingestion, autonomous tools, deployment, or capability claims.
